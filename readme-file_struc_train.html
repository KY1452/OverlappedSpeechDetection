Description of key files and folders list for model training

  • klass-osd the working or current directory for running the kedro project.

  • klass-osd/conf config folders

  • klass-osd/conf/base folder for default configuration settings as foundation for all other configuration environments.

  • klass-osd/conf/base/catalog.yml registry of all data sources used in this project

  • klass-osd/conf/base/credentials.yml config of credentials parameters for mlflow logging

  • klass-osd/conf/base/logging.yml config of parameters for logging

  • klass-osd/conf/base/parameters.yml config of parameters used in kedro project

  • klass-osd/conf/local folder for testing configurations - the files in this folder will overwrite any configurations with the same filename in base folder.

  • klass-osd/conf/README.md inherited from kedro, ignore

  • klass-osd/data folder for data used in kedro project. Detailed data folder and files during kedro run provided at the end of this list.

  • klass-osd/debug.log log file for debug messages generated during running of kedro pipeline.

  • klass-osd/docs inherited from kedro, ignore.

  • klass-osd/error.log log file for error messages generated during running of kedro pipeline.

  • klass-osd/info.log log file for info messages generated during running of kedro pipeline.

  • klass-osd/klass2-conda-env.yml contain the environment requirements for running kedro pipeline.

  • klass-osd/notebooks folder for notebooks.

  • klass-osd/pyproject.toml inherited from kedro, ignore.

  • klass-osd/README.md inherited from kedro, ignore.

  • klass-osd/src/ folder for source code pertaining to kedro.

  • klass-osd/src/klass_osd/__init__.py init file for klass-osd module.

  • klass-osd/src/klass_osd/__main__.py main script for initiating kedro pipeline.

  • klass-osd/src/klass_osd/hooks.py a file used to add certain functionality before and after kedro pipeline runs - configured for mlflow logging in our case.

  • klass-osd/src/klass_osd/pipeline_registry.py centralized location to define and manage all the project’s data pipelines.

  • klass-osd/src/klass_osd/settings.py a file used by kedro to load configs.

  • klass-osd/src/klass_osd/pipelines/create_hashing_dataset_libri2mix_detailed folder for scripts to sample and hash dataset (#2 in pipeline).

  • klass-osd/src/klass_osd/pipelines/data_600ms_chunks folder for scripts to chunk sparselibrimix audio into 600ms chunks with labels (#4-6 in pipeline).

  • klass-osd/src/klass_osd/pipelines/data_chunks folder for scripts to chunk test and evaluation audio into 600ms chunks with labels (#4-6 in pipeline).

  • klass-osd/src/klass_osd/pipelines/data_processing folder for scripts to generate overlapped speech labels from raw data (#1 in pipeline).

  • klass-osd/src/klass_osd/pipelines/data_segment folder for scripts to create segments from train audio with labels (#4-6 in pipeline).

  • klass-osd/src/klass_osd/pipelines/data_validation_training_for_wavLM folder for scripts to validate training and evaluation datasets (#3 in pipeline).

  • klass-osd/src/klass_osd/pipelines/model_prediction folder for scripts to use a model to generate predictions, and metrics for evaluation (#9 in pipeline).

  • klass-osd/src/klass_osd/pipelines/wavlm_hf_dataset folder for scripts to curate the datasets in huggingface format (#7 in pipeline).

  • klass-osd/src/klass_osd/pipelines/wavlm_model_training folder for scripts to train the wavLM model (#8 in pipeline).

  • klass-osd/src/klass_osd/pipelines/*/nodes.py node file which declares functions used in particular * pipeline.

  • klass-osd/src/klass_osd/pipelines/*/pipeline.py declares inputs and outputs, and calls node in * pipeline.

  • klass-osd/src/klass_osd/utils folder for utils.

  • klass-osd/src/klass_osd/utils/data_prep.py helper function used in data_chunks and data_segment pipelines.

  • klass-osd/src/klass_osd/utils/logging.py helper function to carry out mlflow logging.

  • klass-osd/src/pyproject.toml inherited from kedro, ignore.

  • klass-osd/src/requirements.txt inherited from kedro, ignore.

Description of key files and folders list in data folder

  • klass-osd/data/01_raw folder to place raw files for Libri2Mix, SparseLibriMix, and short noises & rirs for augmentations.

  • klass-osd/data/02_intermediate folder to store outputs for data processing steps.

  • klass-osd/data/02_intermediate/hash_libri2/validated/eval_format_check_report.json file to document evaluation files which failed the validation check.

  • klass-osd/data/02_intermediate/hash_libri2/validated/eval_gender_combi_chart.jpeg picture to show gender combination breakdown for evaluation dataset.

  • klass-osd/data/02_intermediate/hash_libri2/validated/eval_snr_histogram_plt.jpeg picture to show snr histogram for evaluation dataset.

  • klass-osd/data/02_intermediate/hash_libri2/validated/hashing_2spk_df.csv final dataset post-validation used for data segments and data chunks.

  • klass-osd/data/02_intermediate/hash_libri2/validated/train_format_check_report.json file to document training files which failed the validation check.

  • klass-osd/data/02_intermediate/hash_libri2/validated/train_gender_combi_chart.jpeg picture to show gender combination breakdown for train dataset.

  • klass-osd/data/02_intermediate/hash_libri2/validated/train_snr_histogram_plt.jpeg picture to show snr histogram for train dataset.

  • klass-osd/data/02_intermediate/hash_libri2_detailed/libri2mix_clean_detailed_datasets_final.csv file contains detailed information like gender combi, SNR, audio arrays for whole Libri2Mix clean dataset.

  • klass-osd/data/02_intermediate/hash_libri2_detailed/libri2mix_modeling_dataset_final.csv file contains hashed detailed dataset with splits for train, eval, test, before data validation.

  • klass-osd/data/02_intermediate/Libri2Mix/eval folder containing chunked files for evaluation dataset.

  • klass-osd/data/02_intermediate/Libri2Mix/test folder containing chunked files for test dataset.

  • klass-osd/data/02_intermediate/Libri2Mix/train folder containing segment files for training dataset.

  • klass-osd/data/02_intermediate/Libri2Mix/wav16k/max/metadata folder containing Libri2Mix dataset labels (output from data processing pipeline).

  • klass-osd/data/02_intermediate/sparselibrimix/clean/sparselibrimix2_600ms_chunks_clean.parquet file containing labelled chunks for sparselibri2mix clean dataset.

  • klass-osd/data/02_intermediate/sparselibrimix/noisy/sparselibrimix2_600ms_chunks_noisy.parquet file containing labelled chunks for sparselibri2mix noisy dataset.

  • klass-osd/data/02_intermediate/sparselibrimix/sparselibrimix2_processed_data.parquet file containing sparselibrimix2 dataset labels (output from data processing pipeline).

  • klass-osd/data/02_intermediate/libri2mix_clean_overall_detailed_dev.csv intermediate file used in create_hashing_dataset_libri2mix_detailed pipeline.

  • klass-osd/data/02_intermediate/libri2mix_clean_overall_detailed_test.csv intermediate file used in create_hashing_dataset_libri2mix_detailed pipeline.

  • klass-osd/data/02_intermediate/libri2mix_clean_overall_detailed_train_100.csv intermediate file used in create_hashing_dataset_libri2mix_detailed pipeline.

  • klass-osd/data/02_intermediate/libri2mix_clean_overall_detailed_train_360.csv intermediate file used in create_hashing_dataset_libri2mix_detailed pipeline.

  • klass-osd/data/02_intermediate/renaming_sparselibrimix.json file which contains “True” if sparselibrimix folders have been renamed.

  • klass-osd/data/03_primary inherited from kedro, not used.

  • klass-osd/data/04_feature inherited from kedro, not used.

  • klass-osd/data/05_model_input folder to store huggingface datasets to be read by model.

  • klass-osd/data/06_models folder to store trained models.

  • klass-osd/data/07_model_output inherited from kedro, not used.

  • klass-osd/data/08_reporting folder to store evaluation results for test sets.
